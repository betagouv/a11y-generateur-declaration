Générateur de déclaration d'accessibilité

La déclaration d’accessibilité est une obligation du Référentiel Général d’Amélioration de l’Accessibilité (RGAA) concernant le service public.

Ce générateur permet d'obtenir une déclaration d’accessibilité conforme que vous pouvez télécharger et affiner si besoin.

  • Informations techniques

Informations générales

Ce sont les informations minimales pour votre déclaration d’accessibilité.

À propos de votre déclaration

Vous pouvez renseigner le nom de votre organisation, l’adresse web et le nom de votre site ou application. Pour les applications mobiles, inclure le numéro de version et la date de sortie permettant d’identifier une version spécifique.

Date de publication

Fournir une date aide les utilisateurs à comprendre si la déclaration d’accessibilité est activement tenue à jour ou si elle est dépassée. Idéalement, la date d’une déclaration d’accessibilité ne devrait pas dépasser un an, sinon elle peut être considérée comme non tenue à jour. La mention du mois rend la date plus claire au niveau international (par exemple, "1er février 2019" au lieu de "01/02/2019", qui est ambigu).

Informations techniques

Dans cette section, vous pouvez fournir des détails techniques supplémentaires pour aider les utilisateurs à comprendre les problèmes qu’ils peuvent observer. Il s’agit notamment d’informations sur la compatibilité avec les navigateurs web et les technologies d’assistance.

Contenus non accessibles

Il existe de nombreuses situations dans lesquelles l’accessibilité peut être limitée. Par exemple, vous pouvez ne pas être en mesure d’assurer l’accessibilité instantanée du contenu généré par l’utilisateur. La transparence dans de telles situations aide les utilisateurs à comprendre tout problème qu’ils peuvent observer et à trouver des solutions de rechange le cas échéant. Selon la loi, les organismes publics (et dont les startups d’états) sont tenus de fournir des informations sur les parties du contenu qui ne sont pas conformes, la raison de cette non-conformité et, le cas échéant, où trouver des alternatives accessibles.

Limites d’accessibilités

Listez les contenus qui ont des limitations d’accessibilité, une description du problème que peut rencontrer un utilisateur, une explication brève de pourquoi le problème peut arriver, et ce qu’il peut faire en attendant, comme par exemple contacter quelqu’un ou utiliser telle ou telle alternative.

Contenus non soumis à l’obligation d’accessibilité

Certaines sections ou contenus non accessibles n’entrent pas dans le champ d’application de la législation appliquable. Il n’y a donc pas d’obligations sur ces contenus. Il est toutefois utile d’indiquer aux utilisateurs les alternatives possibles.

(Exemple : « Le système de cartographie Open Street map est un service externe non soumis à obligation. »)

Compatibilité avec l’environnement de l’utilisateur

Malgré tous les efforts déployés, l’accessibilité peut ne pas fonctionner correctement dans toutes les combinaisons de système d’exploitation, de navigateur web et de technologie d’assistance. Communiquer sur les environnements testés aide l’utilisateur à déterminer si les outils utilisés pourraient être la cause d’un problème observé.

Compatibilité avec l’environnement utilisateur

Décrivez les environnements (combinaisons de navigateurs web, de technologies d’assistance et de systèmes d’exploitation) avec lesquels le contenu est censé fonctionner.

(Exemple : « navigateur X avec la technologie d’assistance Y et le système d’exploitation Z »)

Incompatibilités connues

Aidez les utilisateurs à comprendre quelles versions de systèmes d’exploitation, de navigateurs web et de technologies d’assistance ne sont pas (ou plus) prises en charge. Cela les aide à déterminer s’ils peuvent utiliser votre site web ou votre application mobile avec leur environnement actuel.

Environnements incompatibles

Décrivez les environnements (combinaisons de navigateurs web, de technologies d’assistance et de systèmes d’exploitation) avec lesquels le contenu n’est pas censé fonctionner.

(Exemple : « navigateurs vieux de plus de 3 versions majeures »)

Technologies utilisées

Vous pouvez vous appuyer sur des technologies web spécifiques, telles que JavaScript, WAI-ARIA ou SVG, pour garantir l’accessibilité de votre contenu. Communiquer sur ces technologies indispensables peut aider les utilisateurs à comprendre la raison des problèmes qu’ils peuvent observer.

Décrivez les technologies indispensables pour atteindre la conformité. Le contenu n’est pas conforme si cette technologie est désactivée ou non supportée.
